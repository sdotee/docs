## Project Overview

This is the documentation site for S.EE, a URL shortener service. Built with Next.js 16 and Fumadocs, using Bun as the package manager and runtime.

## Development Commands

```bash
# Install dependencies
bun install

# Run development server (localhost:3000)
bun run dev

# Build for production (runs pre-build script + next build)
bun run build

# Pre-build only (regenerates API docs from OpenAPI spec)
bun run build:pre

# Start production server
bun run start
```

## Architecture

### Content Pipeline
- **MDX Documentation**: `content/docs/**/*.mdx` - Manual documentation pages
- **OpenAPI Generation**: `scripts/generate-docs.ts` generates API docs from `openapi_swagger.yaml` into `content/docs/api/`
- **Source Config**: `source.config.ts` defines Fumadocs MDX configuration

### Key Files
- `lib/source.tsx` - Document loader with OpenAPI integration, custom Lucide icons plugin, and i18n URL formatter
- `lib/openapi.ts` - OpenAPI configuration pointing to `openapi_swagger.yaml`
- `lib/i18n.ts` - Locale config (`en`, `zh-CN`), flag URLs, OG locale map, helpers
- `lib/i18n-ui.tsx` - Fumadocs UI translations per locale
- `lib/layout.shared.tsx` - Locale-aware nav config
- `app/layout.tsx` - Root layout: `<html>`/`<body>`, fonts, scripts, `RootProvider`. Reads `x-locale` header (set by `proxy.ts`) to set `<html lang>`
- `app/docs/[lang]/layout.tsx` - DocsLayout with locale-aware page tree and language switcher
- `app/docs/[lang]/[[...slug]]/page.tsx` - Docs page renderer
- `app/docs/[lang]/llms.mdx/[[...slug]]/route.ts` - Per-locale Markdown export
- `components/language-switcher.tsx` - Locale picker; custom `onChange` builds correct `/docs/[lang]/...` URLs
- `proxy.ts` - Fumadocs i18n middleware with custom URL formatter for the `/docs` prefix; sets `x-locale` response header

**URL structure** (`hideLocale: 'default-locale'`):
- English: `/docs/foo/`
- Other: `/docs/{lang}/foo/` (e.g. `/docs/zh-CN/foo/`, `/docs/ja/foo/`)

The loader's custom `url()` in `lib/source.tsx` keeps locale **after** `/docs` (Fumadocs default would be `/zh-CN/docs/...`).

**Translating manual MDX pages**:
- Create sibling files: `name.{lang}.mdx` next to `name.mdx` (e.g. `signup.zh-CN.mdx`).
- For sidebar/meta translations: `meta.{lang}.json` next to `meta.json`.
- Internal links inside translated MDX must include the locale prefix (e.g. `/docs/zh-CN/signup`, not `/docs/signup`).
- Pages with no translation **fall back to English** (handled in `app/docs/[lang]/[[...slug]]/page.tsx` and the `llms.mdx` route), so partial translations are safe.

**Translating API reference pages** (Scheme A Рђћ frontmatter only):
- API MDX files in `content/docs/api/` are auto-generated by `scripts/generate-docs.ts` from `openapi_swagger.yaml`. Do NOT edit them directly.
- Per-language translations live in `scripts/api-translations/{lang}.json`, keyed by operationId. Each entry has `title` and `description`.
- The script generates `<operationId>.{lang}.mdx` for every non-empty entry, with translated frontmatter pointing to the same English `<APIPage>` body. Empty entries are skipped (page falls back to English).
- After editing translations, run `bun run build:pre` to regenerate.

**Adding a new language**:
1. Add the locale to `languages` in `lib/i18n.ts`, plus entries in `langNameMap`, `ogLocaleMap`, `flagMap`.
2. Add at least `displayName` in `lib/i18n-ui.tsx` (other UI strings fall back to English).
3. Append the locale to the rewrite regex in `next.config.mjs` (the `(en|zh-CN|...)` group, in two places).
4. Create `scripts/api-translations/{lang}.json` (copy any existing one as a template).
5. Translate manual content as `*.{lang}.mdx` and `meta.{lang}.json`.

**Other notes**:
- The proxy matcher (`proxy.ts`) excludes `api-search`, `sitemap.xml`, `llms.txt`, `llms-full.txt`, `og/`, and `_next/`. Add new non-localized routes to that list.
- `<html lang>` is set in the root layout from the `x-locale` header that `proxy.ts` writes on every response.

### Sidebar Icons
Icons in sidebar are configured via `icon` frontmatter in MDX files. The custom `createLucideIconsPlugin()` in `lib/source.tsx` transforms icon names to Lucide React components. Icon names must match exports from `lucide-react`'s `icons` object (e.g., `Home`, `BookOpen`, `CircleQuestionMark`).

## Bun Usage

Always use Bun instead of Node.js/npm:
- `bun install` instead of `npm install`
- `bun run <script>` instead of `npm run <script>`
- `bun <file>` instead of `node <file>`
