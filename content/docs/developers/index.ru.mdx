## Обзор

API S.EE позволяет программно создавать и управлять короткими ссылками, текстовыми публикациями и обменом файлами. Используйте его, чтобы:

- Массово создавать короткие ссылки
- Публиковать текст и фрагменты кода с подсветкой синтаксиса
- Загружать и делиться файлами
- Управлять пользовательскими доменами
- Получать аналитические данные
- Встраивать S.EE в свои приложения

## Начало работы

### Что понадобится

1. Аккаунт S.EE с активным тарифом
2. Токен доступа к API

### Аутентификация

Во всех API-запросах нужен токен доступа в заголовке `Authorization`:

```bash
Authorization: Bearer YOUR_ACCESS_TOKEN
```

---

### Базовый URL

```text
https://s.ee/api/v1
```

## Примеры запросов

### Создать короткую ссылку

```bash
curl -X POST "https://s.ee/api/v1/shorten" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "target_url": "https://example.com/very-long-url",
    "domain": "s.ee",
    "custom_slug": "my-link"
  }'
```

### Создать текстовую публикацию

```bash
curl -X POST "https://s.ee/api/v1/text" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "My Code Snippet",
    "content": "console.log(\"Hello, World!\");",
    "text_type": "source_code",
    "domain": "fs.to"
  }'
```

### Загрузить файл

```bash
curl -X POST "https://s.ee/api/v1/file/upload" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -F "file=@/path/to/your/file.png" \
  -F "domain=fs.to"
```
