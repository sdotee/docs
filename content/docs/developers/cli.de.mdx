Ein Kommandozeilen-Client für die S.EE-Plattform, mit dem Sie Kurz-URLs, Text-Snippets und Dateien direkt aus Ihrem Terminal erstellen und verwalten können.

- [sdotee/cli](https://github.com/sdotee/cli)

## Funktionen

- **Short URL Management**: Erstellen, aktualisieren und löschen Sie Kurzlinks mit benutzerdefinierten Slugs, Domains und Ablaufdaten
- **Text Snippets**: Teilen Sie Code-Snippets und Notizen aus stdin oder Dateien
- **File Upload**: Laden Sie Dateien direkt oder über stdin hoch
- **Domain & Tag Management**: Verfügbare Domains auflisten und Tags verwalten
- **Password Protection**: Schützen Sie Inhalte mit Passwörtern
- **JSON Output**: Maschinenlesbare Ausgabe für Skripting

## Installation

### macOS

```bash
brew tap sdotee/homebrew-tap
brew install see
```

### Debian / Ubuntu

```bash
dpkg -i see-cli_<version>_<arch>.deb
```

### RHEL / Fedora

```bash
rpm -i see-cli_<version>_linux_<arch>.rpm
```

### Arch Linux

```bash
yay -S see-cli
# or pre-built binary
yay -S see-cli-bin
```

### Aus dem Quellcode

```bash
go install github.com/sdotee/cli@latest
```

Vorgebaute Binärdateien und Pakete finden Sie in den [GitHub Releases](https://github.com/sdotee/cli/releases).

## Konfiguration

Setzen Sie Ihren API-Key per Umgebungsvariable oder Flag:

```bash
# Environment variable (recommended)
export SEE_API_KEY="your-api-key"

# Or use the flag
see --api-key "your-api-key" <command>
```

### Konfigurationsoptionen

| Option | Environment Variable | Description |
|--------|---------------------|-------------|
| `--api-key` | `SEE_API_KEY` | Ihr S.EE API-Key (erforderlich) |
| `--base-url` | `SEE_BASE_URL` | Benutzerdefinierte API-Base-URL |
| `--timeout` | `SEE_TIMEOUT` | Request-Timeout |
| `--json` | - | JSON-Ausgabeformat aktivieren |

## Befehle

### Domains & Tags

```bash
# List available domains for short URLs
see domains

# List available domains for file uploads
see file domains

# List your tags
see tags
```

### Kurz-URLs

```bash
# Create a short URL
see url create https://example.com

# Create with custom options
see url create https://example.com \
  --slug my-link \
  --domain s.ee \
  --title "My Link" \
  --password secret \
  --expires 2024-12-31

# Update a short URL
see url update --domain s.ee --slug my-link --title "New Title"

# Delete a short URL
see url delete --domain s.ee --slug my-link
```

### Text-Snippets

```bash
# Create text from stdin
echo "Hello World" | see text create --title "greeting"

# Create text from file
see text create --file ./snippet.txt --title "Code Snippet"

# Create with options
cat script.sh | see text create \
  --title "Shell Script" \
  --domain fs.to \
  --password secret \
  --expires 2024-12-31

# Update text
see text update --domain fs.to --slug abc123 --title "Updated Title"

# Delete text
see text delete --domain fs.to --slug abc123
```

### Datei-Upload

```bash
# Upload a file
see file upload ./image.png

# Upload with custom domain
see file upload ./document.pdf --domain fs.to

# Upload from stdin
cat image.png | see file upload --stdin --filename image.png

# Delete a file
see file delete <delete-hash>
```

## JSON-Ausgabe

Verwenden Sie das Flag `--json` für maschinenlesbare Ausgabe – nützlich für Skripting:

```bash
see url create https://example.com --json | jq '.short_url'
```

## Beispiele

### Kurz-URL erstellen und in die Zwischenablage kopieren (macOS)

```bash
see url create https://example.com --json | jq -r '.short_url' | pbcopy
```

### Screenshot hochladen und URL erhalten (Linux)

```bash
see file upload ~/screenshot.png --json | jq -r '.url'
```

### Kurz-URLs stapelweise erstellen

```bash
while read url; do
  see url create "$url" --json
done < urls.txt
```
