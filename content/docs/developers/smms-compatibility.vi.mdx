S.EE cung cấp một lớp API tương thích với SM.MS, cho phép bạn chuyển từ SM.MS sang S.EE với rất ít thay đổi trong mã. Chỉ cần thay endpoint API là bạn đã có thể sử dụng.

## Chuyển đổi nhanh

Thay base URL API của SM.MS bằng URL tương đương của S.EE:

| SM.MS | S.EE |
|-------|------|
| `https://sm.ms/api/v2` | `https://s.ee/api/v1/file` |

## Endpoint được hỗ trợ

| SM.MS Endpoint | S.EE Endpoint | Description |
|----------------|---------------|-------------|
| `POST /api/v2/upload` | `POST /api/v1/file/upload` | Tải ảnh lên |
| `GET /api/v2/upload_history` | `GET /api/v1/files` | Lấy lịch sử tải lên |

## Tải ảnh lên

### Yêu cầu

```bash
curl -X POST "https://s.ee/api/v1/file/upload" \
  -H "Authorization: your-api-key" \
  -F "smfile=@/path/to/image.png" \
  -F "domain=sm.ms" \
  -F "custom_slug=my-image"
```

### Tham số

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `smfile` | file | Yes | Tệp hình ảnh cần tải lên (alias: `file`) |
| `domain` | string | No | Tên miền tùy chỉnh cho liên kết ngắn |
| `custom_slug` | string | No | Slug tùy chỉnh cho URL (ngẫu nhiên nếu không cung cấp) |

## Lịch sử tải lên

### Yêu cầu

```bash
curl "https://s.ee/api/v1/files" \
  -H "Authorization: your-api-key"
```

### Tham số

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `page` | integer | No | Số trang (mặc định 1, 30 tệp mỗi trang) |

## Khác biệt so với SM.MS

Mặc dù S.EE duy trì khả năng tương thích với API của SM.MS, vẫn có một số cải tiến:

| Tính năng | SM.MS | S.EE |
|---------|-------|------|
| Tên miền tùy chỉnh | Không | Có |
| Slug tùy chỉnh | Không | Có |
| Giới hạn kích thước tệp | 5MB | Khác nhau theo gói |
| Định dạng được hỗ trợ | Chỉ hình ảnh | Hình ảnh, tài liệu và nhiều loại khác |

## Lấy API Key của bạn

Để dùng API của S.EE, bạn cần API key:

1. Đăng ký hoặc đăng nhập tại [s.ee](https://s.ee)
2. Truy cập **[Tools > API Token](https://s.ee/user/developers/)**
3. Tạo một API key mới
