A S.EE fornece uma camada de API compatível com SM.MS, permitindo que você migre do SM.MS para a S.EE com alterações mínimas no código. Basta substituir o endpoint da API e começar a usar.

## Migração rápida

Substitua a URL base da API do SM.MS pelo equivalente da S.EE:

| SM.MS | S.EE |
|-------|------|
| `https://sm.ms/api/v2` | `https://s.ee/api/v1/file` |

## Endpoint compatível

| Endpoint SM.MS | Endpoint S.EE | Descrição |
|----------------|---------------|-------------|
| `POST /api/v2/upload` | `POST /api/v1/file/upload` | Enviar imagem |
| `GET /api/v2/upload_history` | `GET /api/v1/files` | Obter histórico de envios |

## Enviar imagem

### Requisição

```bash
curl -X POST "https://s.ee/api/v1/file/upload" \
  -H "Authorization: your-api-key" \
  -F "smfile=@/path/to/image.png" \
  -F "domain=sm.ms" \
  -F "custom_slug=my-image"
```

### Parâmetros

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|----------|-------------|
| `smfile` | file | Sim | O arquivo de imagem a enviar (alias: `file`) |
| `domain` | string | Não | Domínio personalizado para a URL curta |
| `custom_slug` | string | Não | Slug personalizado para a URL (aleatório se não for informado) |

## Histórico de envios

### Requisição

```bash
curl "https://s.ee/api/v1/files" \
  -H "Authorization: your-api-key"
```

### Parâmetros

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|----------|-------------|
| `page` | integer | Não | Número da página (padrão 1, 30 arquivos por página) |

## Diferenças em relação ao SM.MS

Embora a S.EE mantenha compatibilidade com a API do SM.MS, há algumas melhorias:

| Recurso | SM.MS | S.EE |
|---------|-------|------|
| Domínio personalizado | Não | Sim |
| Slug personalizado | Não | Sim |
| Limite de tamanho de arquivo | 5MB | Varia conforme o plano |
| Formatos compatíveis | Apenas imagens | Imagens, documentos e mais |

## Obter sua chave de API

Para usar a API da S.EE, você precisa de uma chave de API:

1. Cadastre-se ou faça login em [s.ee](https://s.ee)
2. Acesse **[Ferramentas > Token de API](https://s.ee/user/developers/)**
3. Gere uma nova chave de API

## Precisa de ajuda?

Se encontrar algum problema durante a migração, [entre em contato com o suporte](https://s.ee/contact) ou consulte nossa [documentação da API](/docs/pt-BR/developers/api).
