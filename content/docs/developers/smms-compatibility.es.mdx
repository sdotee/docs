S.EE ofrece una capa de API compatible con SM.MS, lo que le permite migrar de SM.MS a S.EE con cambios mínimos de código. Solo tiene que reemplazar el endpoint de la API y estará listo.

## Migración rápida

Reemplace la URL base de la API de SM.MS por su equivalente en S.EE:

| SM.MS | S.EE |
|-------|------|
| `https://sm.ms/api/v2` | `https://s.ee/api/v1/file` |

## Endpoint compatible

| Endpoint de SM.MS | Endpoint de S.EE | Descripción |
|----------------|---------------|-------------|
| `POST /api/v2/upload` | `POST /api/v1/file/upload` | Subir imagen |
| `GET /api/v2/upload_history` | `GET /api/v1/files` | Obtener historial de subidas |

## Subir imagen

### Solicitud

```bash
curl -X POST "https://s.ee/api/v1/file/upload" \
  -H "Authorization: your-api-key" \
  -F "smfile=@/path/to/image.png" \
  -F "domain=sm.ms" \
  -F "custom_slug=my-image"
```

### Parámetros

| Parámetro | Tipo | Obligatorio | Descripción |
|-----------|------|----------|-------------|
| `smfile` | file | Sí | El archivo de imagen que se va a subir (alias: `file`) |
| `domain` | string | No | Dominio personalizado para el enlace corto |
| `custom_slug` | string | No | Slug personalizado para la URL (aleatorio si no se proporciona) |

## Historial de subidas

### Solicitud

```bash
curl "https://s.ee/api/v1/files" \
  -H "Authorization: your-api-key"
```

### Parámetros

| Parámetro | Tipo | Obligatorio | Descripción |
|-----------|------|----------|-------------|
| `page` | integer | No | Número de página (predeterminado 1, 30 archivos por página) |

## Diferencias con respecto a SM.MS

Aunque S.EE mantiene compatibilidad con la API de SM.MS, incluye algunas mejoras:

| Función | SM.MS | S.EE |
|---------|-------|------|
| Dominio personalizado | No | Sí |
| Slug personalizado | No | Sí |
| Límite de tamaño de archivo | 5MB | Varía según el plan |
| Formatos compatibles | Solo imágenes | Imágenes, documentos y más |

## Obtenga su clave API

Para usar la API de S.EE, necesita una clave API:

1. Regístrese o inicie sesión en [s.ee](https://s.ee)
2. Vaya a **[Herramientas > Token de API](https://s.ee/user/developers/)**
3. Genere una nueva clave API
