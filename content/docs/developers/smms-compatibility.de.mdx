S.EE bietet eine mit SM.MS kompatible API-Schicht, sodass Sie mit minimalen Codeänderungen von SM.MS zu S.EE migrieren können. Ersetzen Sie einfach den API-Endpunkt und legen Sie los.

## Schnelle Migration

Ersetzen Sie die SM.MS-API-Basis-URL durch das S.EE-Äquivalent:

| SM.MS | S.EE |
|-------|------|
| `https://sm.ms/api/v2` | `https://s.ee/api/v1/file` |

## Unterstützte Endpunkte

| SM.MS Endpoint | S.EE Endpoint | Description |
|----------------|---------------|-------------|
| `POST /api/v2/upload` | `POST /api/v1/file/upload` | Bild hochladen |
| `GET /api/v2/upload_history` | `GET /api/v1/files` | Upload-Verlauf abrufen |

## Bild hochladen

### Anfrage

```bash
curl -X POST "https://s.ee/api/v1/file/upload" \
  -H "Authorization: your-api-key" \
  -F "smfile=@/path/to/image.png" \
  -F "domain=sm.ms" \
  -F "custom_slug=my-image"
```

### Parameter

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `smfile` | file | Yes | Die hochzuladende Bilddatei (Alias: `file`) |
| `domain` | string | No | Benutzerdefinierte Domain für den Kurzlink |
| `custom_slug` | string | No | Benutzerdefinierter URL-Slug (zufällig, wenn nicht angegeben) |

```bash

  -H "Authorization: your-api-key"
```

## Unterschiede zu SM.MS

Obwohl S.EE die Kompatibilität zur SM.MS-API beibehält, gibt es einige Erweiterungen:

| Funktion | SM.MS | S.EE |
|---------|-------|------|
| Benutzerdefinierte Domain | Nein | Ja |
| Benutzerdefinierter Slug | Nein | Ja |
| Dateigrößenlimit | 5MB | Variiert je nach Tarif |
| Unterstützte Formate | Nur Bilder | Bilder, Dokumente und mehr |

## API-Key abrufen

Um die S.EE API zu verwenden, benötigen Sie einen API-Key:

1. Registrieren Sie sich oder melden Sie sich bei [s.ee](https://s.ee) an
2. Gehen Sie zu **[Werkzeuge > API-Token](https://s.ee/user/developers/)**
3. Generieren Sie einen neuen API-Key
