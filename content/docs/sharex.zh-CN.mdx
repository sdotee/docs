[ShareX](https://getsharex.com/) 是一款适用于 Windows 的开源截图和文件分享工具。S.EE 为 ShareX 提供了自定义上传器配置，因此您可以直接将截图、文件、文本和代码上传到 S.EE，并获得可分享链接。

另请参阅：[ShareX 自定义上传器文档](https://getsharex.com/docs/custom-上传器)

---

## 自定义上传器

S.EE 为以下上传类型提供配置：

| 上传器 | 功能 |
|----------|-------------|
| 短网址上传器 | 通过 S.EE 创建短网址 |
| 文本上传器 | 上传文本片段 |
| 代码上传器 | 上传带语法高亮的代码 |
| Markdown 上传器 | 上传带渲染预览的 Markdown |
| 图片和文件上传器 | 上传图片和文件 |

## 安装

获取下面的 `.sxcu` 配置文件。您可以下载后导入，也可以在 ShareX 中通过 URL 导入。

| 上传器 | 下载 / 导入链接 |
|----------|------------------------|
| 短网址上传器 | [https://assets.seecdn.com/files/sharex/S.EE_URL_Shortener.sxcu](https://assets.seecdn.com/files/sharex/S.EE_URL_Shortener.sxcu) |
| 文本上传器 | [https://assets.seecdn.com/files/sharex/S.EE_Text_Uploader.sxcu](https://assets.seecdn.com/files/sharex/S.EE_Text_Uploader.sxcu) |
| 代码上传器 | [https://assets.seecdn.com/files/sharex/S.EE_Code_Uploader.sxcu](https://assets.seecdn.com/files/sharex/S.EE_Code_Uploader.sxcu) |
| Markdown 上传器 | [https://assets.seecdn.com/files/sharex/S.EE_Markdown_Uploader.sxcu](https://assets.seecdn.com/files/sharex/S.EE_Markdown_Uploader.sxcu) |
| 图片和文件上传器 | [https://assets.seecdn.com/files/sharex/S.EE_File_Uploader.sxcu](https://assets.seecdn.com/files/sharex/S.EE_File_Uploader.sxcu) |

### 导入

1. 打开 **ShareX**
2. 前往 **目标位置** > **自定义上传器设置**
3. 点击 **导入**，然后执行以下任一操作：
   - 选择已下载的 `.sxcu` 文件
   - 直接粘贴下载 URL

## 设置 API 密钥

1. 在 **[工具 > API Token](https://s.ee/user/developers/)** 获取您的 API 密钥
2. 在 ShareX 中打开 **自定义上传器设置**，并选择您导入的 S.EE 上传器
3. 在 **请求头** 区域，将 `Authorization` 的值设置为您的 API 密钥

<Callout type="warn">
不要公开分享您的 API 密钥，也不要将其提交到版本控制系统中。
</Callout>

## 使用方法

### 选择上传器

在 ShareX 中，前往 **目标位置** 并分配 S.EE 上传器：

- 短网址上传器 → S.EE URL Shortener
- 文本上传器 → S.EE Text Uploader、Code Uploader 或 Markdown Uploader
- 图片上传器 → S.EE File Uploader
- 文件上传器 → S.EE File Uploader

### 上传

使用 ShareX 快捷键或右键菜单即可创建短网址、上传截图、分享文本片段等。ShareX 会将内容上传到 S.EE，并把链接复制到您的剪贴板。

## 故障排查

| 问题 | 解决方法 |
|-------|-----|
| 401 错误 | 检查 `Authorization` 请求头中的 API 密钥是否设置正确 |
| 403 错误 | 确认您的 API 密钥具有正确权限 |
| 文件被拒绝 | 文件类型可能不受支持。请查看 [文件分享](/docs/zh-CN/file-sharing) 了解允许的类型 |

您可以在 **自定义上传器设置** 中点击 **测试**，确认配置是否正常工作。
