## 개요

S.EE 파일 공유를 사용하면 짧고 공유하기 쉬운 링크로 파일을 업로드하고 공유할 수 있습니다. 이미지, 문서, 오디오, 비디오 및 기타 파일을 공유할 때 적합합니다.

---

## 파일 업로드

### 빠른 업로드 (드래그 앤 드롭)

1. [https://s.ee/user/dashboard](https://s.ee/user/dashboard) 에 로그인합니다
2. 왼쪽 사이드바에서 **Sharing > Files** 를 클릭하거나 [https://s.ee/user/files/](https://s.ee/user/files/) 로 바로 이동합니다
3. **Drag & Drop** 창에 파일 또는 폴더를 끌어다 놓습니다
   - 창을 클릭해서 파일을 찾아 선택할 수도 있습니다
   - 폴더를 업로드하면 폴더 안의 모든 파일이 함께 업로드됩니다
4. 파일은 기본 도메인과 임의 슬러그로 공유됩니다

### 고급 업로드 (파일 생성)

파일 공유를 더 세밀하게 설정하려면:

1. **파일 생성** 버튼을 클릭합니다
2. 설정을 구성합니다
   - **Domain** - 공유 페이지에 사용할 도메인을 선택합니다
   - **Custom Slug** - 사용자 정의 URL 슬러그를 설정합니다
   - **Other options** - 태그, 만료 등
3. 파일을 선택해 업로드합니다

## 파일 링크

업로드된 각 파일에는 두 가지 링크가 있습니다.

| Link Type | Format | Description |
|-----------|--------|-------------|
| Share Page | `fs.to/[your_slug]` | 도메인과 슬러그를 사용자 정의할 수 있습니다 |
| Direct Link | `i.see.you/[yyyy]/[mm]/[dd]/[random_slug]/[filename].[ext]` | 원래 파일명으로 파일에 직접 접근할 수 있습니다 |

**참고:** 현재 Direct Link 는 사용자 정의 도메인을 지원하지 않습니다.

<Callout type="warn">
실행 파일 (.exe, .msi, .bat, .app 등) 은 보안상의 이유로 **허용되지 않습니다**.
</Callout>

<Callout type="info">
다음 확장자에 대해서는 스크립트 악용을 줄이기 위해 응답 `Content-Type` 을 `text/plain` 으로 강제할 수 있습니다: `.js`, `.mjs`, `.html`, `.htm`, `.xhtml`, `.shtml`, `.svg`, `.xml`, `.json`, `.map`.
</Callout>

<Callout type="info">
알 수 없거나 모호한 파일 형식의 경우 응답 `Content-Type` 을 `application/octet-stream` 으로 강제합니다.
</Callout>

## 파일 보관 기간

- 구독이 활성 상태인 동안 파일은 **영구 보관** 됩니다
- 구독이 만료되면 파일은 삭제 전 **7 일간** 보관됩니다

## FAQ

### 누가 파일을 업로드할 수 있나요?

유료 구독자만 파일을 업로드할 수 있습니다. 유효한 구독이 필요합니다.

### 다운로드 제한이 있나요?

아니요. 공유 파일에는 다운로드 횟수 제한이나 대역폭 제한이 없습니다.

### 공유 페이지 URL 을 사용자 정의할 수 있나요?

네. **파일 생성** 업로드 방식을 사용하면 공유 페이지에 대해 사용자 정의 도메인과 슬러그를 설정할 수 있습니다.

### Direct Link URL 도 사용자 정의할 수 있나요?

현재 Direct Link 는 고정 형식을 사용하며 사용자 정의 도메인을 지원하지 않습니다.
