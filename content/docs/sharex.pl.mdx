[ShareX](https://getsharex.com/) to otwartoźródłowe narzędzie do przechwytywania ekranu i udostępniania plików dla Windows. S.EE udostępnia własne konfiguracje moduł przesyłaniaów do ShareX, dzięki czemu możesz przesyłać zrzuty ekranu, pliki, tekst i kod bezpośrednio do S.EE i od razu otrzymywać link do udostępnienia.

Zobacz też: [dokumentację własnych modułów przesyłania ShareX](https://getsharex.com/docs/custom-uploader)

---

## Własne moduły przesyłania

S.EE ma konfiguracje dla następujących typów przesyłania:

| Uploader | Działanie |
|----------|-------------|
| Skracacz URL | Skraca URL-e przez S.EE |
| Uploader tekstu | Przesyła fragmenty tekstu |
| Uploader kodu | Przesyła kod z podświetlaniem składni |
| Uploader Markdown | Przesyła Markdown z wyrenderowanym podglądem |
| Uploader obrazów i plików | Przesyła obrazy i pliki |

### Importowanie

1. Otwórz **ShareX**
2. Przejdź do **Miejsca docelowe** > **Ustawienia własnego modułu przesyłania**
3. Kliknij **Importuj**, a następnie:
   - wybierz pobrany plik `.sxcu`
   - albo wklej bezpośrednio URL do pobrania

## Skonfiguruj klucz API

1. Pobierz klucz API z **[Narzędzia > Token API](https://s.ee/user/developers/)**
2. W ShareX otwórz **Ustawienia własnego modułu przesyłania** i wybierz importer S.EE
3. W sekcji **Nagłówki** ustaw wartość `Authorization` na swój klucz API

<Callout type="warn">
Nie udostępniaj publicznie klucza API i nie zapisuj go w systemie kontroli wersji.
</Callout>

## Użycie

### Wybierz moduły przesyłania

W ShareX przejdź do **Miejsca docelowe** i przypisz moduły przesyłania S.EE:

- Skracacz URL -> S.EE URL Shortener
- Uploader tekstu -> S.EE Text Uploader, Code Uploader albo Markdown Uploader
- Uploader obrazów -> S.EE File Uploader
- Uploader plików -> S.EE File Uploader

### Przesyłanie

Użyj skrótów klawiaturowych ShareX albo menu kontekstowego, aby skrócić URL, przesłać zrzut ekranu, udostępnić fragment tekstu itd. ShareX prześle dane do S.EE i skopiuje link do schowka.

## Rozwiązywanie problemów

| Problem | Rozwiązanie |
|-------|-----|
| Błąd 401 | Sprawdź, czy klucz API jest poprawnie ustawiony w nagłówku Authorization |
| Błąd 403 | Upewnij się, że klucz API ma odpowiednie uprawnienia |
| Plik odrzucony | Typ pliku może nie być obsługiwany. Zobacz [Udostępnianie plików](/docs/pl/file-sharing) |

Możesz kliknąć **Sprawdź** w **Ustawienia własnego modułu przesyłania**, aby sprawdzić, czy wszystko działa poprawnie.
